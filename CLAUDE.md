# CLAUDE.md - Offline Scrabble (Flutter Web / Vercel)

## Project Overview
A 100% offline, standalone Scrabble game built with Flutter Web and optimized for static deployment on Vercel. The application relies entirely on local memory, asset bundles, and client-side persistence, requiring zero external server handshakes or API network connectivity.

## Core Architectural Boundaries
- **No Cloud Dependencies:** Absolutely no Firebase, AWS, or external database configurations.
- **Local Validation:** Words are checked entirely on-device using a performance-optimized local Prefix Tree (Trie).
- **Static Deployment:** Optimized for compilation into static assets (`flutter build web --release`) to fit Vercel's free-tier infrastructure.
- **State Persistence:** Game state is written locally on every successful turn.

---

## Technical Stack & Dependencies
- **Framework:** Flutter Web (Dart)
- **Web Renderer:** CanvasKit (default for pixel-perfection) or HTML (for faster load speeds)
- **Local Persistence:** `hive` & `hive_flutter` (Lightweight, zero-native binary key-value store)
- **State Management:** `ChangeNotifier` with `AnimatedBuilder` or lightweight `Provider`

---

## Data Models & Structure

### 1. Game State (`GameBoard`)
- Represented as a $15 \times 15$ two-dimensional array.
- Matrix coordinates map to cell types:
  - Standard, DL (Double Letter), TL (Triple Letter), DW (Double Word), TW (Triple Word).
- Each cell contains an optional `Tile` object.

### 2. Tile & Tile Bag
- **Tile:** Holds immutable properties: `char letter`, `int value`, and optional tracking for blank tiles.
- **Tile Bag:** Instantiated with the standard 100-tile distribution. Shuffled locally using the Fisher-Yates algorithmic paradigm.

### 3. Player Profile
- Tracks `int score`, `List<Tile> rack` (maximum capacity of 7), and an `isAI` boolean flag for pass-and-play or local bot extension blocks.

---

## Algorithmic Specifications

### 1. Word Validation Engine (The Trie)
- **Data Structure:** A Prefix Tree (`TrieNode`) instantiated in-memory during the application boot-strapping sequence.
- **Source:** Loaded from a raw text asset containing official dictionary rules (`assets/dictionary.txt`) via `rootBundle.loadString()`.
- **Search Complexity:** $O(L)$, where $L$ is the length of the string under validation. Case-insensitive matching.

### 2. Referee & Scoring Engine (`ScrabbleReferee`)
- **Linear Validation:** Verifies that all newly placed tiles share a single axis (row or column) and form a continuous, contiguous chain connecting to pre-existing board infrastructure.
- **Adjacency & Cross-Word Extraction:** Performs bidirectional layout sweeps across perpendicular vectors to capture intersecting words generated by the current move.
- **Mathematical Evaluation:**
  1. Calculate aggregate base letter values applying cell multipliers (DL, TL).
  2. Compute total sum and apply word multipliers sequentially (DW, TW).
  3. Append a +50 point "Bingo" bonus if the turn consumed all 7 rack tiles.

## UI & Interactivity Layer
- **Layout:** Fixed or constrained $15 \times 15$ grid system utilizing adaptive aspect ratios.
- **Gestures:** Native Flutter `Draggable` (for rack tiles) and `DragTarget` (for cell zones).
- **Rollback Routine:** Automatic UI-state reversion and tile rack bounce-back on invalid or canceled turn inputs.

---

## Local Persistence Schema (Hive)
- **Box Name:** `scrabble_game_state`
- **Keys:** 
  - `board_matrix`: Serialized state snapshot of the active board.
  - `player_pool`: Score profiles and active rack arrays.
  - `bag_state`: Remaining contents of the active tile collection.
- **Trigger Strategy:** Executed automatically inside the state-manager immediately following a valid "End Turn" call.
